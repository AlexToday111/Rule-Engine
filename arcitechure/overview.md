# Обзор

Rule Engine — stateless backend-сервис на Spring Boot для принятия решений по цепочке правил.
Сервис принимает контекст (decisionType, subjectId, attributes), извлекает правила из БД и вычисляет решение.

## Границы системы

Внутри сервиса:
- REST API для оценки решений и CRUD правил.
- Движок вычисления условий и эффектов.
- Кэширование набора правил.
- Метрики через Micrometer + Actuator.

Вне сервиса:
- PostgreSQL как хранилище правил.
- Клиенты, вызывающие API (скоринг, антифрод, eligibility и т.д.).

## Технологии

- Spring Boot (REST, валидация, конфигурация)
- Spring Data JPA
- Flyway миграции
- Micrometer + Actuator (Prometheus endpoint)
- PostgreSQL

## Основные свойства

- Детерминированная обработка правил по приоритету.
- Набор правил хранится в БД, изменения мгновенно влияют на новые запросы.
- Кэш правил для снижения нагрузки на БД (TTL 30 секунд).
